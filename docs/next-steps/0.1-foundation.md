# 0.1 Foundation — implementation plan

This is the living plan for milestone **0.1 Foundation** in [`../../ROADMAP.md`](../../ROADMAP.md). It is allowed (and expected) to drift as we learn. Edit it; don't leave stale entries.

The goal of 0.1 is the *skeleton*: a multi-platform SwiftUI shell that compiles on iOS and tvOS, with a mock library and a working AVPlayer prototype. No real network calls.

When 0.1 ships, a fresh contributor should be able to:

```bash
git clone …
cd Aether
xcodegen generate
open Aether.xcodeproj
# run on iOS 26 simulator → see mock library → play sample video
# run on tvOS 26 simulator → see mock library → play sample video
```

…and feel that this is going to become a premium Apple-platform app.

---

## Implementation order

The order matters — earlier items unblock later ones.

1. **App bootstrap** — `project.yml` generates a working multi-platform shell.
2. **Package setup** — `AetherCore` package wired into both app targets.
3. **Design system foundation** — tokens, type, spacing, color, a couple of components.
4. **Mock data** — fixture-driven media library so the app has something to show before any networking.
5. **Navigation** — Home → Detail → Player flow on both platforms.
6. **Playback prototype** — AVPlayer plays a sample video; resume state writes to memory.
7. **Polish pass** — empty states, focus states, transitions baseline.

Each one should land as its own PR.

---

## Dependencies between tasks

```
project.yml ──► Package setup ──► Design system ──► Navigation
                                       │                │
                                       └──► Mock data ──┘
                                                        │
                                                        ▼
                                              Playback prototype
                                                        │
                                                        ▼
                                                Polish pass
```

- The design system can start before navigation, but navigation should not start before tokens exist.
- Mock data can be built in parallel with the design system; they meet at the Home screen.
- Playback can't start until navigation can reach a detail screen.

## Tasks

### App bootstrap

- Author `project.yml` with `Aether` (iOS) and `Aether-tvOS` (tvOS) app targets sharing source where appropriate.
- iOS 26 / tvOS 26 deployment targets.
- SwiftUI app lifecycle (`@main` App struct).
- Generated project file is *not* checked in.
- README documents the `xcodegen generate` step.

**Done when:** `xcodegen generate && xcodebuild -scheme Aether -destination 'platform=iOS Simulator,name=iPhone 17'` produces an .app.

### Package setup

- Add `AetherCore` as a local Swift package inside the repo.
- One library product (`AetherCore`) consumed by both app targets.
- Internal module layout matches `docs/architecture/ARCHITECTURE.md`:
  `Models`, `MediaSources`, `Playback`, `Downloads`, `Storage`, `DesignSystem`.
- Strict concurrency enabled at the package level.

**Done when:** both app targets `import AetherCore` and link cleanly.

### Design system foundation

- `DesignSystem/Tokens` — colors, type styles, spacing, radii, motion durations.
- `DesignSystem/Components` — start small: `CardView`, `SectionHeader`, `BackdropImage`, `CachedAsyncImage` stub.
- One reusable preview helper so every component has a SwiftUI preview.
- Tokens documented in [`../ux/DESIGN_PRINCIPLES.md`](../ux/DESIGN_PRINCIPLES.md) as numbers, not just adjectives.

**Done when:** the Home screen and the Detail screen can be built without hard-coding a single color, font, or spacing value.

### Mock data

- A small JSON fixture in `Aether/Resources/MockLibrary.json` with ~10 movies + 1 TV show with 6 episodes.
- A `MockMediaSource` in `AetherCore/MediaSources/` that loads the fixture and returns `MediaItem` values.
- One sample video URL (Apple's public stream is fine for now).
- Resume points are an in-memory `Dictionary` for 0.1.

**Done when:** the Home screen renders rows of artwork from the fixture; tapping/focusing into one shows a Detail screen with metadata.

### Navigation

- `NavigationStack` on iOS; `NavigationStack` (or whatever feels right on tvOS) on tvOS — adapter at the app target.
- Routes: `home → detail(MediaItem) → player(MediaItem)`.
- Back navigation works correctly on both platforms.
- Deep-linkable in principle — routes are types, not strings.

**Done when:** end-to-end traversal works on both simulators with no hangs and no view-state leakage.

### Playback prototype

- A `PlaybackSession` actor in `AetherCore/Playback/` with the minimum surface: `prepare(item:)`, `play()`, `pause()`, `seek(to:)`, `currentState async`.
- A SwiftUI `PlayerView` in `Aether/` that observes session state.
- AVPlayer wraps the sample URL; PiP / AirPlay are not configured yet, but the seam is there.
- Resume position written to memory every 5s.

**Done when:** pressing "Play" from Detail starts playback in a full-screen player on both platforms, with working pause/seek and a visible timeline.

### Polish pass

- One designed empty state (home, no library — even though we always have mock data, this is the path users will see first when real sources land).
- Focused state on tvOS cards is intentional (subtle lift, no type reflow).
- Transition from Detail → Player is a crossfade, not a stock push.
- App icon placeholder set, launch storyboard / SwiftUI launch surface configured.

**Done when:** a stranger using the app on Apple TV without instructions would describe it as "polished."

## Recommended first issues

These are good starting tickets and can be opened on day one.

1. **`project.yml` + multi-platform shell** — `architecture`, `ios`, `tvos`
2. **`AetherCore` package skeleton** — `architecture`
3. **DesignSystem: tokens + first components** — `ux`
4. **MockMediaSource + Home screen rendering** — `architecture`, `ux`
5. **Navigation: Home → Detail → Player** — `architecture`, `ios`, `tvos`
6. **PlaybackSession actor + PlayerView (iOS)** — `playback`, `ios`
7. **PlayerView (tvOS) + focus polish** — `playback`, `tvos`, `ux`
8. **Empty states + Detail → Player crossfade** — `ux`

Each maps to one of the tasks above. Each is a single PR.

## Notes for future me

- Don't be tempted to add real Plex / Synology calls before navigation feels good with mock data. The mock pass is the chance to nail the *experience*. Networking is easy; experience is not.
- Don't fight SwiftUI navigation. If something feels hard, the design probably wants adapting more than the code does.
- Resist adding any third-party dependency before 0.2.
