# Movie Detail Redesign — Unified Library Phase 2d

> **Status:** design brief (not for Phase 1). Implement **after** Unified Library
> Phase 1 + 2a–2c land, as Phase 2d. Cross-platform.
> **One line:** the Detail screen should feel like Apple TV / Infuse, not a media-
> server config page — and it should represent a **`UnifiedMediaItem`**, not a
> Plex or Jellyfin item.

The screen works functionally today but reads as a configuration page (stacked
disclosure rows). Goal: the user instantly sees **what it is, how to play it,
where it's available, and the quality** — without feeling like they're
configuring a server. Reduce density, increase hierarchy, keep advanced controls
secondary.

This builds on the unified model already shipped (`UnifiedMediaItem` /
`UnifiedSource` / `preferredSource`). DetailView currently takes a `MediaItem`;
this phase moves it to a `UnifiedMediaItem` (one title, multiple sources behind
it). The user should think *"this movie is available here,"* not *"this belongs
to Plex."*

---

## Layout (top → bottom)

Most users should rarely scroll past Sources.

1. **Hero** — backdrop, poster, title, year, runtime, genre + **technical badges**.
2. **Primary actions** — Resume / Play / Restart.
3. **Available Sources** — preferred source + override.
4. **Playback** — Audio / Subtitles / Quality (compact rows, bottom-sheets).
5. **Overview** — summary.
6. **Media Information** — codecs / resolution / bitrate / HDR (only when known).

---

## 1. Hero + technical badges

Keep backdrop, poster, title, year, runtime, genre. **Add compact chips** below
the metadata when the data is available (from `MediaInfo` on the preferred
source):

```
[4K] [HDR] [Dolby Vision] [Atmos]      or      [1080p] [H.264] [AC3 5.1]
```

Badges are display-only, derived from `MediaInfo` (videoResolution, isHDR,
isDolbyVision, codecs, audio channels). A new small `AetherBadge` chip in the
design system. Show only the badges we actually have.

## 2. Primary actions

Only the most important actions are immediately visible:

```
[ Resume ]                 (primary, when a resume point exists)
[ Play From Beginning ]    (secondary)
```

**Download must not compete with playback.** It is *not* a primary button. When
downloaded, the offline state shows in the Sources / download area as
`Available Offline`, not a Download button up top.

## 3. Available Sources (the unified heart)

A dedicated section listing every source behind the title, with the preferred one
called out and **manual override**:

```
Preferred Source
  Offline                        ← or Plex / Jellyfin

Available Sources
  ✓ Plex            4K HDR · Server name
    Jellyfin        1080p · Server name
    Offline         12.4 GB
```

- Preferred follows the priority (offline → plex → jellyfin → emby) but the user
  can **override** for this playback — important for quality comparison and
  troubleshooting. The override routes playback through the chosen
  `UnifiedSource.item` via the existing `MediaSource.resolvePlayback`.
- Each row shows server name + quality (`UnifiedSource.serverName` / `.quality`).
- Future: Emby appears here with no special treatment.

## 4. Playback (secondary, compact)

Keep the existing Audio / Subtitles / Quality rows — but group them under a
**Playback** section, compact rows with the current value, opening bottom-sheet
selectors (no large pickers inline):

```
Playback
  Audio        English 5.1
  Subtitles    Czech
  Quality      Original
```

These already exist (`AetherDisclosureRow` + bottom sheets); this is grouping +
de-emphasis, scoped to the active source.

## 5. Overview

The summary, as today.

## 6. Media Information (advanced, conditional)

A new section, **only shown when data is available**, for advanced users:

```
Media Information
  Video Codec   HEVC
  Audio Codec   EAC3
  Resolution    4K
  Bitrate       24 Mbps
  HDR           Dolby Vision
```

Read-only, from `MediaInfo` of the active/preferred source.

## Download state

- **Downloaded:** `✓ Available Offline` + storage size (e.g. `12.4 GB`). No
  Download button.
- **Not downloaded:** `Download` + `Estimated Size 12.4 GB`, secondary placement.

---

## Visual direction

Current Aether branding. **Add no new controls or settings** — this is
re-organisation + de-densification. Apple TV + Infuse + Aether, not a Plex
settings screen.

## Architecture notes

- `DetailView` takes a `UnifiedMediaItem`; the unified surfaces (Home/Search)
  already navigate via the preferred `MediaItem` today, so this phase switches
  navigation to pass the unified item and resolves the active source within.
- Playback / download still go through the chosen `UnifiedSource.item` +
  existing `MediaSource` — no new playback path.
- Badges + Media Information are pure display off `MediaInfo`; no new fetching.
