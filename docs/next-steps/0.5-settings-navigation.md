# Navigation + Settings Dashboard — Unified Library Phase 4

> **Status:** design brief (not for Phase 1). Implement as Phase 4, after the
> unified read surfaces. Cross-platform (iPhone / iPad / tvOS / visionOS).
> **One line:** drop Storage as a top-level tab (downloads are just a source),
> and turn Settings from a list into a **premium dashboard**.

---

## 1. Storage removal + navigation

Downloads are part of the media ecosystem now (an **Offline source** in Unified
Library), not a separate area. Remove **Storage** from primary navigation on
**all** platforms.

**New tab set (all platforms):**

```
Home · Library · Discover · Search · Settings
```

- **Discover** becomes first-class everywhere (today it's tvOS-only).
- **Search** becomes a first-class tab (today it's an inline `.searchable` on
  Home/Library).
- Download/storage management moves into **Settings → Downloads**:
  - Downloaded Movies, Downloaded Shows, Storage Usage, Download Quality,
    Clear Downloads.
- Offline content surfaces through Unified Library — `Available Offline`,
  `Downloaded` rail, the Offline source — not a dedicated Storage tab.

`RootTabView` already branches tabs per platform; this generalises it (and the
existing `StorageView` content largely moves under Settings → Downloads).

## 2. Settings as a dashboard (not a list)

Settings should feel like a media-platform dashboard. **Don't just centre the
existing list on iPad** — use a **split layout** that uses the space. Same
concept on every large surface; the phone keeps a single column.

**Sections (left / nav):** Accounts · Sources · Downloads · Playback ·
Appearance · About.

### iPad / tvOS — split dashboard

```
┌──────────────────────┬─────────────────────────────┐
│      AETHER           │         What's New          │
│ Accounts             │ Version 0.5.0              │
│ Sources              │ • Unified Library          │
│ Downloads            │ • Discover                 │
│ Playback             │ • Cinema V1                │
│ Appearance           │  Upcoming                  │
│ About                │ • Cinema V2 · Emby · SMB   │
│ Version 0.5.0        │  Connected Sources         │
│                      │ Plex       Online          │
│                      │ Jellyfin   Online          │
│                      │ Offline    128 GB          │
└──────────────────────┴─────────────────────────────┘
```

- **tvOS** has lots of unused space today — same left-nav / right-dashboard
  concept (What's New · Roadmap · Connected Sources).

### visionOS — spatial dashboard

Left panel: Accounts · Sources · Downloads · Playback · Appearance · About.
Right panel cards:

```
Connected Sources              Storage Summary
  Plex Server     Online         Downloads    128 GB
  Jellyfin Server Online         Free Space   512 GB
  Offline Library Available      Download Status …
```

Feels like a native spatial dashboard, not a settings form.

## 3. Source health in Settings

Settings exposes **source health/status** (Plex Online / Jellyfin Online /
Offline available + size). Library exposes *media*; Settings exposes *sources*.
This separation matters — the user never thinks about where content originates
while browsing.

## 4. What's New (interactive version row)

The version row becomes tappable → opens a **What's New** modal with the
changelog highlights for the version. No separate changelog screen; each release
reuses the pattern.

```
Version 0.5.0  →   What's New
                   0.5.0
                   • Unified Library
                   • Discover
                   • Download integration
                   • Playback improvements
```

## 5. Settings cleanup (carry-over)

Remove "Planned"/marketing rows (Transcoding Planned, Offline Downloads
Planned) — those features exist. Surface the existing Appearance store as a
**System / Dark / Light** selector.

---

## Unified Library framing

The whole redesign assumes the source model **Offline · Plex · Jellyfin ·
(future) Emby**, with no source getting special UI treatment. Goal: Settings
feels like a premium media-platform dashboard — Apple TV + Infuse + Aether — not
a traditional config menu.
