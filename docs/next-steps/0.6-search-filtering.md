# Search & Filtering — proposed 0.6

> **Status:** design brief (not started). Builds directly on Unified Library
> (0.5.0) — filtering operates on `UnifiedMediaItem` metadata so it behaves
> identically regardless of source.
>
> **One line:** richer, source-agnostic filtering (audio/subtitle language,
> video, audio format, source) behind a compact Apple-TV/Infuse-style filter
> UI — *not* a settings-form.

## 1. Advanced media filters

Unified Library gives us per-source metadata, so Search and Library filtering
should expand well beyond content type. Dimensions:

- **Audio language** — Czech / English / German / Japanese / … (e.g. "all movies
  with Czech audio").
- **Subtitle language** — Czech / English / … plus **Forced** and **SDH / CC**.
- **Video** — 4K · 1080p · HDR · Dolby Vision · HEVC · H.264.
- **Audio format** — Stereo · 5.1 · 7.1 · Dolby Atmos · DTS · DTS-HD MA · TrueHD.
- **Source** — Offline · Plex · Jellyfin · Emby (future).

**Examples to support:** all Czech-audio movies · all Czech-subtitle movies ·
all downloaded movies · all 4K HDR movies · all Atmos movies · Czech audio **and**
Atmos · all Plex movies with Dolby Vision.

## 2. Matching semantics

Filter over `UnifiedMediaItem`: a title matches if **any one of its sources**
satisfies **all active dimensions** (so "Czech audio AND Atmos" means a *single*
source has both — the one you'd actually play). Within a dimension, multiple
selections are **OR** (Czech *or* English audio); across dimensions, **AND**.

Filter options shown in the UI are **derived from the loaded result set** — only
surface languages/codecs/resolutions actually present, never an empty filter.

## 3. UI direction

Compact, like Apple TV / Infuse / Netflix — **not** a settings-style form.

```
Search
[ Filters ⌄ 2 ]        ← button with active-count badge

  Audio Language
  Subtitle Language
  Video
  Audio Format
  Source
```

A **Filters** button opens a sheet with **expandable sections + chips**. Apply
filters the result grid in place.

## 4. Surfaces

Decided scope for the first cut:

- **Search tab** — primary; loads result pages into memory, so client-side
  filtering is complete and accurate.
- **Library** — also filter the Library browse/grid. *(Caveat: Library is
  single-source + paginated today; until it's unified, filters only apply to
  already-loaded items. Revisit when Library goes unified.)*

## 5. Implementation outline (when picked up)

- **AetherCore `MediaFilter`** — value type holding the selected sets per
  dimension, with `matches(item:kind:)` (single source) and
  `matches(_:UnifiedMediaItem)` (any-source). Plus `MediaFilterOptions(items:)`
  to extract the present options for the UI. Pure + unit-tested.
- **Capability extraction** from `MediaItem`:
  - audio/subtitle language ← `audioTracks[].languageCode` /
    `subtitleTracks[].languageCode`; forced ← `subtitleTracks[].isForced`.
  - audio format ← `audioTracks[].channels` (2 = stereo, 6 = 5.1, 8 = 7.1) +
    `.codec` (best-effort match for Atmos / DTS / DTS-HD MA / TrueHD).
  - video resolution / HDR / Dolby Vision / video codec ← `mediaInfo`.
- **Filter UI** — `AetherFilterBar` (button + badge) + a chip sheet, reused by
  Search and Library.
- Slices: **A** model + tests → **B** UI + Search → **C** Library.

## 6. Known constraints / decisions

- **Video filters are Plex-only at first.** `MediaInfo` (resolution / HDR /
  Dolby Vision / codec) is populated for Plex list items but **not** for
  Jellyfin — Jellyfin returns the data in its `MediaStreams` (Height / Codec /
  VideoRange) but Aether doesn't map it to `mediaInfo` yet. So 4K/HDR/DV/codec
  filters won't match Jellyfin content until a **Jellyfin `mediaInfo` backfill**
  is added. Audio / subtitle / source filters work everywhere today. The UI
  should make this honest (don't imply Jellyfin titles were filtered out on a
  video filter — or do the backfill first).
- **SDH / CC** isn't modelled (`MediaSubtitleTrack` has no SDH flag); needs a
  flag or a title heuristic.
- **Atmos** isn't a clean signal in our data (often EAC3/TrueHD without an
  explicit flag); detection is best-effort on codec/title strings.
- Plex **list rails sometimes omit `Media.Part.Stream`**, so audio/subtitle
  tracks can be missing on un-hydrated list items; detail hydration guarantees
  them. Filtering is best-effort on whatever the list carried.

## Follow-on: Jellyfin video metadata backfill

A prerequisite for universal video filters (and useful on its own for the
Detail badges): map Jellyfin's video `MediaStream` to `MediaInfo`
(Height → resolution, `Codec` → videoCodec, `VideoRange` / `VideoRangeType` →
HDR / Dolby Vision). Once done, video filters become source-agnostic like the
rest.
