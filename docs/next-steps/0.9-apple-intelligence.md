# AI Recommendations — Apple Intelligence concierge (0.9)

Status: **Proposed** · Target release theme: `0.9 "Apple Intelligence"` (after 0.8.x stabilization + TestFlight readiness)

Natural-language recommendations over the user's own library: "recommend a horror", "something scary but not too gory, under 2 hours, like The Shining" → Aether searches the unified library and recommends a real title the user can play, fully on-device.

## Why

- Differentiates Aether from Plex/Infuse/Jellyfin clients and reinforces the **premium, Apple-native, privacy-first** positioning.
- Runs entirely **on-device and offline** via the iOS 26 Foundation Models framework — no cloud LLM, no data leaves the device. This is the *only* recommendation approach consistent with our privacy-first principle.
- Reuses what we already have: `UnifiedLibrary` already exposes rich, structured metadata (`genres`, `overview`, `communityRating`, `tmdbRating`, `year`, `type`) across Plex / Jellyfin / Emby / Local / SMB.

## Guiding principle — library-grounded, no hallucination

**The LLM is never the source of truth about the catalog.** `UnifiedLibrary` is.

The model only:
1. translates natural language into a structured query, and
2. picks and explains a recommendation **from a candidate shortlist, by item ID**.

It can never recommend a title that isn't in the library, because it only ever sees (and selects from) items the deterministic engine hands it. This is the single most important correctness guardrail — it eliminates the classic failure mode of LLM recommenders inventing titles the user doesn't own.

## Phase status

| Phase | Scope | Platforms | Depends on Apple Intelligence? |
|---|---|---|---|
| **P1** | `RecommendationEngine` (deterministic core) + App Intents / Shortcuts / Spotlight | iOS · iPadOS · macOS · visionOS · **tvOS** | No — ships everywhere |
| **P2** | `RecommendationConcierge` (Foundation Models: NL parse + grounded ranking) | iOS · iPadOS · macOS · visionOS | Yes — gated, graceful fallback to P1 |
| **P3** | UI surfaces ("Ask Aether", Discover hero CTA, tvOS rail) | per-platform (see UX table) | Mixed |
| **P4** | Polish — localized reasons (en/cs/uk), quality telemetry, fallback copy | all | — |

P1+P2 deliver value across **all** platforms without depending on Apple Intelligence; P2's "AI magic" is an enhancement layer built on the tested P1 core, so the AI risk is isolated.

## Architecture (4 layers)

```
┌─ Entry points (per platform) ───────────────────────────────┐
│  Siri / Shortcuts · Spotlight · in-app "Ask Aether" · tvOS tile │
└───────────────┬─────────────────────────────────────────────┘
                ▼
┌─ App Intents (app target) ──────────────────────────────────┐
│  RecommendMediaIntent  +  AetherShortcuts (AppShortcutsProvider) │
└───────────────┬─────────────────────────────────────────────┘
                ▼
┌─ AI layer (gated: iOS/iPadOS/macOS/visionOS) ───────────────┐
│  RecommendationConcierge — wraps LanguageModelSession           │
│   • NL → @Generable query   • rank + reason shortlist (by ID)   │
└───────────────┬─────────────────────────────────────────────┘
                ▼
┌─ RecommendationEngine (AetherCore — shared, deterministic) ──┐
│  filter / rank over UnifiedLibrary → candidates. RUNS ON tvOS. │
└───────────────┬─────────────────────────────────────────────┘
                ▼
        UnifiedLibrary.unifiedItems(kind:)  ← single source of truth
```

### Layer responsibilities

- **`RecommendationEngine`** — new, in `AetherCore`. Pure, `Sendable`, no UIKit, no LLM. Lives alongside `HomeFeedBuilder`, reads `UnifiedLibrary.unifiedItems(kind:)`.
  - Input: `RecommendationRequest { genres: [String], mood: String?, maxRuntime: Duration?, type: MediaItem.Kind?, excludeWatched: Bool }`
  - Output: ranked `[UnifiedMediaItem]` (candidate shortlist, ~15 items).
  - Ranking baseline: genre/type/runtime match → sort by rating (`tmdbRating` ?? `communityRating`), de-prioritize already-watched. Deterministic and unit-testable; this is the cross-platform core (incl. tvOS).
- **`RecommendationConcierge`** — new, in app target, behind `#if canImport(FoundationModels)` + runtime availability check. All Foundation Models usage lives here and nowhere else.
  - NL → structured query: `session.respond(to: query, generating: RecommendationQuery.self)` where `RecommendationQuery` is `@Generable` (`@Guide`-annotated genres / mood / maxMinutes / type).
  - Grounded pick: given the engine's shortlist (id + title + overview + rating), `session.respond(generating: Pick.self)` where `Pick` is `@Generable { itemID: String, reason: String }`. The prompt constrains selection to the provided IDs.
  - Availability via `SystemLanguageModel.default.availability` (verify exact API at implementation).
- **`RecommendMediaIntent` + `AetherShortcuts`** — new, app target. Exposes the feature to Siri / Spotlight / Shortcuts. Parameter: free-text query (or a `genre` enum for the simple deterministic path). Result presents a recommendation snippet with a **Play / Resume** action that reuses the existing playback entry path.

## Data flow — one query ("recommend a horror, not too gory, under 2h")

1. **Entry** → `RecommendMediaIntent` receives the text.
2. **Capability check** → is Foundation Models available?
   - **Yes** → `Concierge` parses NL into `@Generable RecommendationQuery` (genres, mood, maxMinutes).
   - **No** (tvOS / older devices) → simple keyword→genre parser. *Same engine downstream.*
3. **Candidates (Swift, deterministic)** → `RecommendationEngine` filters `UnifiedLibrary` → shortlist (~15) of (id, title, overview, rating).
4. **Selection** →
   - **With LLM** → model receives **only the shortlist** and returns `Pick { itemID, reason }`, selecting **only from provided IDs**.
   - **Without LLM** → take top candidate by rating.
5. **Result** → card with the pick + one-line reason + **Play / Resume** button.

Entirely on-device and offline.

## Platform availability & the tvOS exception

**Apple Intelligence / Foundation Models do not run on tvOS** (Apple TV 4K / A15 is not an Apple-Intelligence device), and Siri voice input on tvOS is limited. Therefore:

- The deterministic `RecommendationEngine` lives in `AetherCore` and runs on **all** platforms, including tvOS.
- The LLM layer (P2) is **iOS / iPadOS / macOS / visionOS only**.
- On tvOS the feature is surfaced as **in-app "Smart Recommendations"** (a "For tonight" / "Surprise me" rail or button), **not** as "Siri AI" — to avoid implying a capability Apple does not provide on TV.

| Surface | iOS/iPadOS | macOS | visionOS | tvOS |
|---|---|---|---|---|
| "Ask Aether" conversational field (in Search) | ✅ with AI | ✅ with AI | ✅ with AI | ⚠️ keyword only |
| Siri / Spotlight / Shortcuts | ✅ strong | ⚠️ weaker | ✅ | ⚠️ limited |
| Discover hero CTA ("Find me something for tonight") | ✅ | ✅ | ✅ | ✅ (deterministic) |
| tvOS "For tonight" rail / tile | – | – | – | ✅ (engine, no voice) |

Primary in-app home: **"Ask Aether"** field in Search; secondary: a CTA on the Discover hero (shipped 0.8.5).

## Graceful degradation

A single `RecommendationConcierge.isAvailable` flag drives the branch. Engine + UI are identical across paths; only the LLM mid-step disappears. One code path to maintain, with an optional "smart" step — not two parallel implementations.

## Build order (→ issues)

1. **`RecommendationEngine` in AetherCore** + Swift Testing suite. Deterministic, cross-platform, immediately testable. Value for tvOS too. *(start here)*
2. **App Intents + AppShortcuts** (`RecommendMediaIntent`) wired to the engine = **P1 ship** (works on all devices, no Apple Intelligence dependency).
3. **`RecommendationConcierge`** (Foundation Models, gated) — NL parse + grounded pick = **P2**.
4. **UI surfaces** — "Ask Aether" in Search, Discover hero CTA, tvOS "For tonight" rail.
5. **Polish** — localize reasons (String Catalogs en/cs/uk), recommendation-quality telemetry, fallback/empty-state copy.

## Open questions / risks

- **Device gating** — must detect Apple-Intelligence availability cleanly and always provide the deterministic fallback. Test on a non-AI device.
- **Model context budget** — on-device context is small; never feed the whole library. Pre-filter in Swift, hand the model only a shortlist. (Engine already does this.)
- **Cross-source consistency** — run over `UnifiedLibrary` (all sources), not a single connector.
- **Reason quality / tone** — reasons must be short, accurate, and localizable; guard against the model over-promising ("the scariest film ever").
- **First-run model download** — Foundation Models assets may need download; handle the "model preparing" state.

## Out of scope (for 0.9)

- Cloud LLMs (contradicts privacy-first).
- Cross-device handoff (compute on iPhone/Mac, play on Apple TV) — interesting future note, **not** v1.
- Personalized taste profiles / learning over time — later iteration.
- Conversational multi-turn refinement — start single-shot; revisit after telemetry.
